**日期：** 2026-07-01  
**版本：** v1.0

---

## 1. 项目概述

### 1.1 目标

创建一个所有人都能轻松上手的影像组学 Web 应用，降低影像组学研究的技术门槛。

### 1.2 用户群体

- **初级用户：** 临床医生、研究生，只需上传影像和 ROI 即可自动化提取常规影像组学特征
- **高级用户：** 科研人员，需要精细控制滤波方式、预处理参数、自定义特征等

### 1.3 核心价值

- **零门槛：** 无需编程，无需安装软件，浏览器打开即用
- **低配置友好：** 所有计算在云端完成，用户设备无要求
- **可视化验证：** 导入后先检查 ROI 位置，确保数据质量
- **完整链路：** 从数据导入到统计分析，一站式解决

## 3. 功能设计

### 3.1 支持的影像模态

- CT（计算机断层扫描）
- MRI（磁共振成像）
- PET（正电子发射断层扫描）
- 其他 DICOM 格式影像

### 3.2 ROI 输入方式

1. **DICOM RTSTRUCT：** 放疗结构文件，包含多个 ROI
2. **NIfTI 掩模：** `.nii` 或 `.nii.gz` 格式的二值掩模
3. **在线手动勾画：** 在网页上直接绘制 ROI（可选功能）

### 3.3 特征提取类型

#### 3.3.1 标准 PyRadiomics 特征

- **形状特征（14个）：** 体积、表面积、紧凑度等
- **一阶统计量（18个）：** 均值、方差、偏度、峰度等
- **纹理特征：**
  - GLCM（灰度共生矩阵）
  - GLRLM（灰度游程长度矩阵）
  - GLSZM（灰度大小区域矩阵）
  - GLDM（灰度距离矩阵）
  - NGTDM（邻域灰度差矩阵）

#### 3.3.2 深度学习特征

- 使用预训练网络（ResNet50、VGG16 等）提取高维特征
- 支持 ROI 区域的特征向量提取

#### 3.3.3 自定义特征

- 用户可定义数学表达式，基于已有特征组合新特征
- 例如：`log(feature1 + feature2)`、`feature1 / feature2`

### 3.4 图像预处理选项（高级模式）

#### 3.4.1 重采样

- 体素间距调整（如 `[1.0, 1.0, 1.0]` mm）
- 插值方法选择（线性、最近邻、B 样条）

#### 3.4.2 灰度离散化

- Bin width（默认 25）
- Bin count 模式

#### 3.4.3 归一化

- 启用/禁用
- 归一化范围（如 `[-3σ, 3σ]`）

#### 3.4.4 滤波器

- **内置滤波器：**
  - Original（原始影像）
  - LoG（高斯拉普拉斯，σ = 1.0, 2.0, 3.0）
  - Wavelet（小波变换：LLH, LHL, LHH, HLL, HLH, HHL, HHH）
  - Square（平方）
  - SquareRoot（平方根）
  - Exponential（指数）
  - Logarithm（对数）
- **自定义卷积核：** 用户上传自定义滤波器核

### 3.5 结果输出

#### 3.5.1 数据导出

- CSV 格式（通用）
- Excel 格式（带格式化）
- 特征矩阵（样本 × 特征）

#### 3.5.2 可视化报告

- 特征分布直方图
- 箱线图（不同 ROI 对比）
- 相关性热力图
- PCA 降维可视化

#### 3.5.3 统计分析模块

- **ICC（组内相关系数）：** 一致性分析
- **LASSO 特征筛选：** 自动选择最优特征子集
- **相关性分析：** 特征间相关性矩阵
- **描述性统计：** 均值、标准差、中位数等

## 4. 用户界面设计

### 4.1 工作流程

**5 步流程：**

1. **导入数据** → 上传 DICOM 影像和 ROI 文件
2. **可视化验证** → 检查 ROI 位置、名称是否正确
3. **参数设置** → 选择特征类型、滤波器（初级用户跳过此步）
4. **特征提取** → 显示进度条
5. **结果分析** → 查看结果、下载报告

## 5. 数据流

### 5.1 数据上传流程

```
用户上传 DICOM 文件夹
    ↓
pydicom 解析 DICOM 序列
    ↓
转换为 SimpleITK Image 对象
    ↓
保存到临时目录（服务器本地）
```

### 5.2 ROI 处理流程

```
用户上传 RTSTRUCT / NIfTI
    ↓
解析 ROI 结构（名称、轮廓/掩模）
    ↓
转换为二值掩模（SimpleITK）
    ↓
与影像配准（如需要）
    ↓
保存掩模到临时目录
```

### 5.3 特征提取流程

```
用户选择 ROI + 参数
    ↓
图像预处理（重采样、归一化、滤波）
    ↓
PyRadiomics 提取标准特征
    ↓
PyTorch 提取深度学习特征（可选）
    ↓
自定义特征计算（可选）
    ↓
汇总为特征矩阵（pandas DataFrame）
    ↓
返回结果
```

### 5.4 结果输出流程

```
特征矩阵（DataFrame）
    ↓
├─→ 导出 CSV/Excel
├─→ 生成可视化图表
├─→ 统计分析（ICC、LASSO 等）
└─→ 生成 PDF 报告（可选）
```

## 6. 关键技术决策

### 6.1 为什么选择 Streamlit？

- **快速开发：** Python 生态，无需前端技能
- **易于部署：** Streamlit Cloud 免费托管
- **用户友好：** 用户无需安装任何软件
- **可扩展：** 支持自定义组件

### 6.2 为什么不用纯前端（WebAssembly）？

- **开发成本：** WASM 编译复杂，需要重写算法
- **兼容性：** 难以保证与 PyRadiomics 完全一致
- **维护成本：** 算法更新需要同步更新 WASM 版本
- **性能：** 对于大影像，服务器端计算更稳定

### 6.3 可视化验证为什么是必须步骤？

- **数据质量：** 确保 ROI 位置正确，避免错误结果
- **用户信心：** 让用户看到数据，增加信任
- **错误预防：** 早期发现数据问题，避免浪费时间

### 6.4 如何处理大影像？

- **分块处理：** 大影像分块加载到内存
- **临时存储：** 使用服务器本地临时目录
- **清理机制：** 会话结束后自动清理临时文件

## 10. 限制与注意事项

### 10.1 Streamlit Cloud 限制

- **内存限制：** 免费套餐有内存限制（约 1GB）
- **计算时间：** 长时间计算可能超时
- **并发：** 同时在线用户数有限

### 10.2 数据安全

- **临时存储：** 用户上传的文件存储在服务器临时目录
- **自动清理：** 会话结束后自动删除
- **隐私建议：** 不建议上传敏感患者数据（使用公开数据集测试）

### 10.3 计算性能

- **大影像：** 大尺寸影像（>500MB）可能较慢
- **多 ROI：** 多个 ROI 同时提取需要较长时间
- **深度学习：** DL 特征提取需要 GPU（Streamlit Cloud 不支持）
